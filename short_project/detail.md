## 3. API 리스트 (30분)

> 표 대신 간단 리스트. 요청/응답은 핵심 필드만.

### 인증
- `POST /api/auth/signup` — body: `{ email, password, nickname }` → `{ id, email, nickname }`
- `POST /api/auth/login` — body: `{ email, password }` → `{ accessToken }`

### [도메인2, 예: 게시글]
- `GET /api/posts` — query: `page, size` → `{ posts: [...], totalCount }`
- `POST /api/posts` — body: `{ title, content }` → `{ id, title, content, createdAt }`
- `GET /api/posts/:id` → `{ id, title, content, author, createdAt }`
- `PUT /api/posts/:id` — body: `{ title, content }` → `{ id, title, content }`
- `DELETE /api/posts/:id` → `{ success: true }`

### 공통 규칙
- 인증 헤더: `Authorization: Bearer {token}`
- 에러 응답 공통 형식: `{ "error": { "code": "...", "message": "..." } }`
- 에러 코드: 400(입력오류) / 401(인증실패) / 403(권한없음) / 404(없음)
